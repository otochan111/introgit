git管理
管理対象のフォルダに移動してgit bash（macの場合はターミナル）からgit initとすると.gitができる。
これがリポジトリで、変更履歴などを保存する実態のファイル群。
これが最初のスタート地点。

ファイル変更
↓
(git add)
↓
ステージングエリア（準備室みたいなもの）
↓
(git commit)
↓
ローカルリポジトリに反映
↓
(git push)
↓
リモートリポジトリに反映

・ファイルを変更したときにコミットするファイルを選択するためにあるのがステージングエリア
　つまりステージングエリアにないとコミット対象外となる。
　いろいろな変更をしたときに１作業ずつ指定する。

　git add ファイル名
　git add .　←いまある変更をすべて追加するとき

・変更にメッセージを付けてリポジトリに記録する操作がコミット。
　これによりリポジトリ内に変更が時系列で記録される。
　コミットごとにメッセージを残せる。
　なので１つの作業まとまり単位に対してメッセージを残すと考えたら、コミットは一定の作業まとまりごとに行うのが良い。
　そうすると、何をやったかメッセージ見たらすぐに把握できるようになる（あくまで理想だけど）。

ファイルを削除するときは、
　git rm ファイル名
これで一旦ステージングエリアに行くので、そこからgit commitすることでリポジトリに削除が反映される。
ファイルもちゃんと消える。

＊この時点でステージングエリア上の取り消しが不明。
＊ロールバックの指定方法が不明。
＊pullしてきてプロジェクトに参加する方法が不明。
＊ブランチ切ってちゃんとプロジェクトっぽく進める方法が不明。

Gitコミットしてからの差分を見る場合は、
　git status
を使う。

On branch master　←マスターブランチ使っとるよ
Your branch is ahead of 'origin/master' by 4 commits.　←githubにプッシュされてからローカルリポジトリは4つコミットされてるよ
